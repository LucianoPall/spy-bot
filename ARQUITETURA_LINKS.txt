🧠 LÓGICA INTERNA - Fluxo de Análise
═══════════════════════════════════════════════════════════════════════════════

Request: POST /api/spy-engine
   │
   ├─ 1. Validar URL do anúncio
   │
   ├─ 2. Detectar Niche (Automático)
   │    ├─ Emagrecimento (keywords: emagrecer, dieta, peso, etc)
   │    ├─ Renda Extra (keywords: renda, ganhar, dinheiro, etc)
   │    ├─ iGaming (keywords: cassino, aposta, jogo, etc)
   │    ├─ Estética (keywords: pele, beleza, lifting, etc)
   │    └─ E-commerce (keywords: loja, compre, produto, etc)
   │
   ├─ 3. Tentar Apify (Scraper)
   │    ├─ Se sucesso: usar dados reais do site
   │    └─ Se falha: usar Mock Inteligente
   │
   ├─ 4. Usar Mock Fallback (Se Apify falhar)
   │    ├─ Gera copy realista por niche
   │    ├─ Seleciona imagem de alta qualidade
   │    └─ Transparente ao usuário (sem erros)
   │
   ├─ 5. Enviar para OpenAI (Análise)
   │    ├─ Prompt customizado por niche
   │    ├─ GPT 4 (ou modelo configurado)
   │    └─ Gera análise detalhada
   │
   ├─ 6. Salvar em Supabase
   │    ├─ Tabela: user_ad_analyses
   │    ├─ Tabela: supabase_logs
   │    └─ Row-Level Security: ativo
   │
   └─ 7. Retornar Response
      └─ JSON com: copy, image, niche, analysis

Tabelas Principais:

├─ id: uuid

user_credits (Billing)
├─ user_id: uuid → auth.users

├─ last_updated: timestamp

user_ad_analyses (Histórico)
├─ id: uuid

├─ ad_url: text

├─ copy: text

└─ RLS: ✅ Ativo

├─ id: uuid

├─ user_id: uuid

├─ created_at: timestamp
└─ RLS: ✅ Ativo
